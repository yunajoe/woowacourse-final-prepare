# 12월 이벤트 플래너

## 기능 요구 사항

- 고객들이 식당에 방문할 날짜와 메뉴를 선택한다.
- 이벤트 플래너가 1)주문 메뉴, 2)할인 전 총 주문 금액, 3)증정 메뉴, 4)헤택 내역, 5)총 혜택 금액, 6)할인 후 예상 결제 금액, 7)12월 이벤트 배지 내용 을 보여준다.

### 입력 요구 사항

1. 식당에 방문할 날짜와 메뉴

- 방문할 날짜는 1이상 31 이하의 숫자로만 받는다

2. 주문하실 메뉴와 갯수

- 메뉴는 한번에 최대 20개
  (e.g. 시저샐러드-1, 티본스테이크-1, 크리스마스파스타-1, 제로콜라-3, 아이스크림-1의 총개수는 7개)
- 음료만 주문할 수 없다.(메뉴메뉴 or 메인메뉴 + 음료)
- 해산물파스타-2,레드와인-1,초코케이크-1 와 같은 형식으로 입력한다.

### 입력 요구 사항 예외

1. 식당에 방문할 날짜와 메뉴

- 1 이상 31 이하의 숫자가 아닌 경우, "[ERROR] 유효하지 않은 날짜입니다. 다시 입력해 주세요."라는 에러 메시지를 보여 주기

2. 주문하실 메뉴와 갯수

- 메뉴판에 없는 메뉴를 입력하는 경우
- 메뉴의 개수는 1 이하의 숫자만 입력한 경우
- 메뉴 형식이 예시와 다른 경우 (메뉴 형식은 "해산물파스타-2, 레드와인-2")
- 중복 메뉴를 입력한 경우
- 음료만 주문을 할 수 없다

### 출력 요구 사항

1. 주문 메뉴

- 형식은 메뉴 이름 메뉴갯수 (타파스 1개)
- 출력 순서는 상관이 없다

2. 할인 전 총 주문 금액

- 메뉴 곱하기 갯수

3. 증정 메뉴

- 할인 전 총주문 금액이 12만 원 이상일 때, 샴페인 1개 증정
- 증정 이벤트에 해당하지 않는 경우, 증정 메뉴 "없음"으로 보여 주기

4. 헤택 내역

- 할인 전 총주문 금액 10,000원 이상부터 이벤트가 적용이 된다.
- 할인은 크게 4가지가 있다.
- 크리스 마스 디데이 할인은 2023.12.1 ~ 2023.12.25 까지, 그 외 이벤트는 2023.12.1 ~ 2023.12.31 동안 적용
  4-1) 크리스마스 디데이 할인

  - 1,000원으로 시작하여 크리스마스가 다가올수록 날마다 할인 금액이 100원씩 증가
  - 총주문 금액에서 해당 금액만큼 할인
    (e.g. 시작일인 12월 1일에 1,000원, 2일에 1,100원, ..., 25일엔 3,400원 할인)

4-2) 평일 할인(일~목)

- 평일에는 디저트 메뉴를 메뉴 1개당 2,023원 할인

4-3) 주말 할인(금,토)

- 주말에는 메인 메뉴를 메뉴 1개당 2,023원 할인

4-4) 특별할인

- 이벤트 달력에 별이 있으면 총주문 금액에서 1,000원 할인

5. 총혜택 금액

- 할인 금액의 합계 + 증정 메뉴의 가격

6. 할인 후 예상 결제 금액

- 할인 전 총 주문 금액 - 할인 금액

7. 12월 이벤트 배지

- 총 혜택 금액에 따라 부여된다.
- 5천원 이상 별, 1만원 이상 트리, 2만원 이상 산타
- 이벤트 배지가 부여되지 않는 경우, "없음"으로 보여 주기

## 프로그래밍 요구 사항

- ApplicationTest 성공
- indent(인덴트, 들여쓰기) depth를 3이 넘지 않도록 구현한다. 2까지만 허용
  예를 들어 while문 안에 if문이 있으면 들여쓰기는 2이다.
  힌트: indent(인덴트, 들여쓰기) depth를 줄이는 좋은 방법은 함수(또는 메서드)를 분리하면 된다.
- else를 지양한다
- 도메인 로직에 단위 테스트를 구현
- 사용자가 잘못된 값을 입력할 경우 throw문을 사용해 예외를 발생시킨다. 그런 다음, "[ERROR]"로 시작하는 에러 메시지를 출력하고 해당 부분부터 입력을 다시 받는다.
- @woowacourse/mission-utils에서 제공하는 Console API를 사용하여 구현해야 한다.
  사용자의 값을 입력 받고 출력하기 위해서는 Console.readLineAsync, Console.print를 활용한다
- 입력과 출력을 담당하는 객체를 별도로 생성 (inputview, outputview)

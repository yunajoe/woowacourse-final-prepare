# 점심 메뉴 추천

## 기능 요구 사항 정리

1. 입력 기능 요구 사항

1-1) 코치 이름 입력

- 토미,제임스,포코 와 같이 입력 (쉼포(,)로 구분한다.)
- 코치 이름은 최소 2글자 최대 4글자
- 코치는 최소 2명, 최대 5명

1-2) 코치가 못 먹는 메뉴 입력

- 각 코치는 최소 0개 최대 2개의 못 먹는 메뉴가 있다
- 우동,사리 와 같이 입력 (,로 구분해서 입력)
- 없는 경우에는 빈 값 입력

2. 출력 기능 요구 사항

2-1) 카테고리

- 각 코치에게 한 주에 같은 카테고리는 최대 2회까지
- Randoms.pickNumberInRange(1, 5) 를 사용하여 카테고리를 정한다 (즉, 같은 숫자는 최대 2회)
- 1은 일식, 2은 한식, 3은 중식, 4는 아시안 5는 양식

2-2) 메뉴

- 각 코치에게 한 주에 중복된 메뉴는 안된다.
- Random.shuffle()을 통해 임의의 순서로 섞은 후, 첫 번째 값을 사용해야 한다. (const menu = Randoms.shuffle(menus)[0])
- Randoms.shuffle() 메서드의 인자로 전달되는 메뉴 데이터는, 최초에 제공한 목록을 그대로 전달해야 한다.
- 코치에게 추천할 메뉴를 정할 때 이미 추천한 메뉴, 먹지 못하는 메뉴도 포함된 리스트를 전달해야 한다.
- 추천할 수 없는 메뉴인 경우 다시 섞은 후 첫 번째 값을 사용해야 한다.

## 프로그래밍 요구 사항 정리

- 추천할 수 없는 카테고리인 경우(같은 카테고리가 이미 2개 나온 상황)
